# 소스트리에서 Git LFS 사용 안내

## 현재 상태
✅ Git LFS 설정 완료
✅ 모든 비디오 파일이 Git LFS로 추적 중
✅ `.gitattributes` 파일 생성 완료

## 소스트리에서 확인 방법

### 1. 소스트리 새로고침
- 소스트리를 열고 **새로고침(F5)** 버튼을 클릭하세요
- 또는 **보기 > 새로고침** 메뉴 사용

### 2. 변경사항 확인
소스트리에서 다음 파일들이 보여야 합니다:
- `.gitattributes` (새 파일)
- `public/videos/*.mp4` (수정된 파일, 11개)

### 3. Git LFS 파일 확인
소스트리에서 Git LFS 파일은 일반 파일처럼 보이지만, 실제로는 LFS 포인터로 관리됩니다.

## 소스트리에서 커밋 및 푸시

### 방법 1: 소스트리 GUI 사용
1. **스테이징 영역**에서 변경된 파일 확인
2. 커밋 메시지 입력: `Add video files with Git LFS`
3. **커밋** 버튼 클릭
4. **푸시** 버튼 클릭

### 방법 2: 터미널 사용 (소스트리 내장 터미널)
소스트리에서 **보기 > 터미널 표시** 메뉴를 사용하여 터미널을 열고:

```bash
git commit -m "Add video files with Git LFS"
git push origin master
```

## 주의사항

⚠️ **소스트리에서 Git LFS 파일 크기 표시**
- 소스트리에서는 Git LFS 파일의 실제 크기가 아닌 포인터 파일 크기(약 100바이트)만 표시될 수 있습니다
- 이는 정상입니다. 실제 파일은 Git LFS 서버에 저장됩니다

⚠️ **푸시 시간**
- Git LFS 파일은 일반 파일보다 푸시 시간이 더 걸릴 수 있습니다
- 특히 큰 파일(100MB 이상)은 시간이 걸릴 수 있으니 기다려주세요

## 문제 해결

만약 소스트리에서 Git LFS가 제대로 작동하지 않는다면:

1. **소스트리 재시작**
2. **Git LFS 재설정** (터미널에서):
   ```bash
   git lfs install
   ```
3. **파일 다시 추가**:
   ```bash
   git add public/videos/*.mp4
   ```

## 확인 명령어

터미널에서 다음 명령어로 확인할 수 있습니다:
```bash
# Git LFS 추적 중인 파일 확인
git lfs ls-files

# Git LFS 상태 확인
git lfs status
```
